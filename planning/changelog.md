# changelog.md — Plan Change History

Full detail for every plan change in this project. Maintained by Jarvis via the 3-part plan-change protocol (see `.claude/agents/jarvis.md`).

**A plan change** = a session decision that alters scope, architecture, or approach.

**NOT a plan change** = ship events (a planned feature getting built), bug fixes, refactors, threshold tweaks within an existing approach.

When a real plan change occurs, three things happen:
1. Full entry appended here (newest first under `## Entries`)
2. One-line row added to PLAN.md Plan Changelog table
3. Inline supersession callout added in PLAN.md next to the affected section (preserving original verbatim)
